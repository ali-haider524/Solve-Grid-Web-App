# SolveGrid V5 Mobile QA

Test these widths in Chrome DevTools before deployment:

- 320px
- 360px
- 375px
- 390px
- 412px
- 768px
- 1024px

## Required checks

1. Header shows only logo and hamburger below 840px.
2. Hamburger drawer opens, closes by backdrop and Escape, and contains all categories and calculator links.
3. No horizontal page scrolling appears on tools, finance calculators, graphing, matrices, statistics, or unit conversion.
4. Graphing page displays the graph before equation controls on phone widths.
5. Equation Solver and Statistics Calculator show one compact select instead of stacked workflow cards on phone widths.
6. Unit Converter shows the category select and hides the long category grid on phone widths.
7. Percentage Calculator displays one selected mode, compact inputs, and result panel without a ten-card menu.
8. Matrix grids and statistics tables scroll inside their cards only.
9. Inputs and selects stay at 16px or above to avoid unwanted mobile browser zoom.
10. All new finance calculators start from safe neutral values and explain estimates without implying lender-specific outcomes.
